| Token | Value | Usage |

## Component Contract: CalculationPanel

### Structure
```
<CalculationPanel>
  ┌──────────────────────────────────────┐
  │ [▼] CALCUL D'IMPACT              h-8 │  ← accordion header, border-t border-white/[0.08]
  ├──────────────────────────────────────┤
  │ [État vide — aucun canal sélectionné] │
  │  ou                                   │
  │ [État: pas de profil chargé]          │
  │  ou                                   │
  │ [Inputs + résultats]                  │
  └──────────────────────────────────────┘
```

### State 1 — Empty (no canal selected)
- Height: `h-10` (40px)
- Text: `text-xs text-gray-500 italic` — "Sélectionnez un canal pour calculer son impact"
- Aligned center, matches ElevationPanel empty state pattern

### State 2 — No elevation data loaded
- Height: `h-10` (40px)
- Text: `text-xs text-amber-400` with `AlertCircle size=12`
- Copy: "Chargez le profil altimétrique pour calculer"
- This state exists because calculation requires `ElevationProfile` from the store

### State 3 — Inputs + Results (active state, canal selected + profile loaded)

**Input row layout:**
```
Largeur   [____50____] m     Profondeur  [___5___] m
```
- Row: `flex gap-4 px-4 py-3`
- Each input group: `flex flex-col gap-1`
- Label: `text-[11px] text-gray-400` — "Largeur" / "Profondeur"
- Input: `w-20 h-8 px-2 text-[13px] text-white bg-gray-800 rounded border border-white/[0.12] focus:border-blue-500 focus:outline-none`
- Unit suffix: `text-[12px] text-gray-400 self-end mb-[6px]` — "m"
- Default values displayed on mount: 50 (width), 5 (depth)
- Input type: `type="number"` with `min="0.1"` and `step="1"`

**Results section — order: Volume → ΔSL → % IPCC → Coût**

Each result row:
- Container: `px-4 py-1 flex flex-col gap-[2px]`
- Label: `text-[11px] text-gray-500 uppercase tracking-wider` — e.g. "VOLUME"
- Value: `text-[13px] font-semibold text-white` — e.g. "[0.012 – 0.014] km³"
- Interval brackets are part of the value string: format `[X – Y] unité`

**IPCC indicator (CALC-04):**
- Progress bar container: `mt-1 h-1.5 w-full rounded-full bg-gray-700`
- Progress bar fill: `h-1.5 rounded-full bg-blue-500 transition-all duration-300`
- Fill width: clamped to 100% max via `Math.min(percentMax, 100)%`
- Caption below bar: `text-[11px] text-gray-400 mt-1` — "N–M% du rythme annuel IPCC (4,5 mm/an)"

**Terrain breakdown (part of Coût section):**
- `text-[11px] text-gray-500 mt-1`
- Format: "Plaine X% · Mixte Y% · Montagne Z%"
- Shown only when breakdown is non-trivially mixed (not 100% plain)

**Partial impact section (CALC-05 — conditional):**
- Shown only when `profile.uphillSegments.length > 0`
- Separator: `border-t border-white/[0.06] mt-2 pt-2`
- Label: `text-[11px] text-amber-400 uppercase tracking-wider` — "IMPACT PARTIEL"
- Caption: `text-[11px] text-gray-400` — "Si arrêté au km X :"
- ΔSL partial: same value format `[Y – Z] mm`
- Percentage note: `text-[11px] text-gray-500` — "X% du tracé réalisable"

### Accordion Behavior
- `isOpen` state: local React `useState(true)`
- Auto-opens on canal selection: `useEffect(() => { if (selectedCanalId) setIsOpen(true) }, [selectedCanalId])`
- ChevronDown icon rotates 180° when open: `transition-transform duration-200`
- No animation on panel body (consistent with ElevationPanel — static show/hide)

### Positioning in SidePanel
- Inserted after `<ElevationPanel />` as `{/* Section 5 — Calcul d'impact (accordéon) */}`
- Before the footer `div`
- Same border pattern: `border-t border-white/[0.08]`

## Number Formatting Contract

All displayed intervals follow this exact format: `[min – max] unit`

- Em dash (`–`, U+2013) as separator — not hyphen
- Space on both sides of dash
- Brackets always included — visually enforces UX-01 (scientific honesty)

### Specific format rules:

| Metric | Format | Example |
|--------|--------|---------|
| Volume | `[X.XXX – X.XXX] km³` | `[0.012 – 0.014] km³` |
| ΔSL | `[X.XXX – X.XXX] mm` | `[0.033 – 0.039] mm` |
| ΔSL (small) | `[X.XXe-N – X.XXe-N] mm` — only if < 0.001mm | rare edge case |
| % IPCC | `N.N – M.M% du rythme annuel` | `0.7 – 0.9% du rythme annuel` |
| Coût (< 1000 M€) | `[X – Y] M€` | `[250 – 500] M€` |
| Coût (≥ 1000 M€) | `[X.X – Y.Y] Md€` | `[1.2 – 3.5] Md€` |
| Longueur partielle | `km X.X` | `km 47.3` |

Number locale: `fr-FR` via `Intl.NumberFormat` for thousands separators on large cost values.

Source: RESEARCH.md Pitfall 6, CONTEXT.md decisions, CLAUDE.md scientific honesty principle.

## Interaction Contracts

### Input behavior
- Debounce: none — `useMemo` in `useCalculation.ts` handles perf. Recalculate on every change.
- On blur with empty field: restore default value (50 for width, 5 for depth)
- On invalid (negative, NaN): show inline error, hide result rows
- Keyboard: Tab cycles width → depth → (next focusable)
- All inputs: `type="number"`, `min="0.1"`, `step="1"`, no spinner arrows (CSS: `[appearance:textfield]`)

### Accordion behavior
- Click header: toggle `isOpen`
- Auto-open when canal selected: yes (matches ElevationPanel)
- No animation (static show/hide — consistent with existing panels)

### IPCC bar interaction
- Static display — no tooltip, no hover state
- Bar fills proportional to `percentMax / 100`, capped at 100%
- Color always blue-500 regardless of percentage value

### Stop marker (map)
- Non-interactive: no click handler on marker
- Removed on canal deselection
- Removed when canal becomes fully gravity (profile reloaded)

## Accessibility Contract

- Accordion button: `aria-expanded={isOpen}`, `focus-visible:ring-2 focus-visible:ring-blue-400 focus-visible:ring-offset-2 focus-visible:ring-offset-gray-900`
- Numeric inputs: `aria-label="Largeur du canal en mètres"` / `aria-label="Profondeur du canal en mètres"`
- Error messages: `role="alert"` on inline validation error `div`
- Result values: wrapped in `<dl>` / `<dt>` / `<dd>` pairs for semantic structure
- IPCC bar: `role="progressbar"` with `aria-valuenow={percentMid}` `aria-valuemin={0}` `aria-valuemax={100}` `aria-label="Comparaison IPCC"`
- Color not used as sole differentiator: labels always present alongside color cues
- Contrast: white on gray-800 input = 15.3:1 (exceeds WCAG AA 4.5:1)
