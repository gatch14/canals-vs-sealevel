# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Map Shell + Tracé. Generated by gsd-ui-researcher.
> This is a scientific GIS desktop tool — aesthetics are minimal and functional, not consumer-app.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — pure Tailwind CSS + custom components |
| Icon library | lucide-react |
| Font | system-ui / sans-serif stack (no custom font in v1) |

**Rationale:** CONTEXT.md explicitly excludes shadcn and any component library. Tailwind CSS with custom components is the declared approach. lucide-react is the standard icon library for React + Vite projects with zero bundle overhead concerns at this scale.

## Layout Contract

| Zone | Specification | Source |
|------|--------------|--------|
| Map canvas | 100vw × 100vh, z-index 0, position absolute | CONTEXT.md decisions |
| Side panel | 320px fixed width, position fixed, right: 0, top: 0, height: 100vh, z-index 10 | CONTEXT.md decisions |
| Panel background | semi-transparent dark (#1a1a2e at 95% opacity with backdrop-blur-sm) | derived from dark theme |
| Panel border-left | 1px solid rgba(255,255,255,0.08) | derived from dark theme |
| Mode indicator bar | 32px tall strip at top of panel, full width, color changes per mode | derived from interaction spec |

**Panel sections (extensible for future phases):**
1. Header strip — "Canal Explorer" title + active mode badge
2. Action zone — "Dessiner canal" button + mode state
3. Canal list — scrollable list of traced canals (empty state when none)
4. Footer — reserved for global stats (Phase 6)

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, marker size hint |
| sm | 8px | Button internal padding (vertical), list item gaps |
| md | 16px | Panel horizontal padding, button internal padding (horizontal) |
| lg | 24px | Section padding within panel, gap between panel sections |
| xl | 32px | Header zone height, mode indicator zone |
| 2xl | 48px | Major section breaks inside panel |
| 3xl | 64px | Not used in Phase 1 |

Exceptions:
- Touch targets: N/A (desktop-only tool, v1)
- Map markers: 12px diameter circles (not on 8-point scale, constrained by MapLibre circle-radius spec in pixels)

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Canal list items, info text, tooltips |
| Label | 12px | 400 | 1.4 | Section headers, badges, mode indicator text, stat labels |
| Heading | 16px | 600 | 1.25 | Panel title "Canal Explorer", canal names |

**Two weights only:** 400 (regular) for Body + Label, 600 (semibold) for Heading. Display role (20px/700) deferred to Phase 4 when ΔSL values are shown.

**Font stack:** `ui-sans-serif, system-ui, -apple-system, sans-serif`
No custom font loading — zero network dependency, instant render, matches the no-backend constraint.

All text on dark backgrounds uses `text-white` or `text-gray-300` — never `text-gray-500` or lighter (insufficient contrast on dark panels).

## Interaction Contract

### Mode State Machine

```
SELECTION (default)
  → click "Dessiner canal"  → DRAWING
  → click canal on map      → SELECTED

DRAWING
  → single click on map     → adds waypoint (minimum 2 to finalize)
  → double-click on map     → finalizes canal → SELECTION
  → press Escape            → cancel, discard in-progress → SELECTION
  → click "Annuler"         → cancel, discard in-progress → SELECTION

SELECTED
  → click empty map area    → SELECTION (deselect)
  → click delete in panel   → delete confirmation → SELECTION
  → click "Dessiner canal"  → DRAWING (deselects current)
```

### Cursor Behavior

| Mode | Cursor CSS |
|------|-----------|
| SELECTION | `cursor-default` |
| DRAWING | `cursor-crosshair` (applied to map canvas container) |
| SELECTED | `cursor-pointer` on canal features |

### Hover/Focus States

- Canal list item: `bg-gray-700` on hover (from `bg-gray-800` base)
- "Dessiner canal" button: `bg-blue-600` on hover (from `bg-blue-500` base), `scale-[0.98]` on active
- Delete button: `bg-red-600` on hover (from transparent/ghost base)
- All interactive elements: `focus-visible:ring-2 focus-visible:ring-blue-400 focus-visible:ring-offset-2 focus-visible:ring-offset-gray-900`

### MapLibre Event Bindings

| Event | Handler |
|-------|---------|
| `map.on('click')` | In DRAWING mode: add waypoint; in SELECTION: deselect |
| `map.on('dblclick')` | In DRAWING mode: finalize canal |
| `map.on('mousemove')` | In DRAWING mode: update preview line to cursor position |
| `map.on('mouseenter', 'canals-layer')` | Show hover state (5px width), change cursor to pointer |
| `map.on('mouseleave', 'canals-layer')` | Restore normal state (3px width) |
| `map.on('click', 'canals-layer')` | Select canal |

**MapLibre double-click conflict:** Must call `map.doubleClickZoom.disable()` when entering DRAWING mode and re-enable on exit. This is Pitfall-level — failure causes accidental zoom instead of canal finalization.
