# SEED-001: Modèle économique intégré + écosystèmes générés

## L'idée

Le canal ne fait pas que baisser la montée des eaux — il crée une chaîne de valeur complète :

1. **Eau douce** (dessalement) → zones habitables → population nouvelle → valeur foncière
2. **Sel récupéré** (saumure RO) → sel alimentaire/industriel (~$30-80/tonne)
3. **Minéraux** (magnésium, potassium, calcium) → engrais agricoles → cultures sur place
4. **Spiruline** (algue halophile, eau salée + soleil intense) → marché alimentaire mondial
5. **Aquaculture** (poissons d'eau de mer dans le bassin terminal) → protéines locales
6. **Terres agricoles** : eau douce + engrais sur place → agriculture dans l'ancien désert
7. **Tout ça en prime de** : réduction de la montée des eaux

## Ce que l'app devrait calculer

- **ROI global** : coût construction vs revenus cumulés sur X années
- **Timeline habitabilité** : dans combien d'années la zone devient habitable ?
- **Production spiruline** : tonnes/an estimées selon surface bassin terminal
- **Production sel** : tonnes/an selon volume dessalé
- **Surface agricole potentielle** : km² cultivables autour des nœuds
- **Valeur économique totale** : €/an générés par le système complet
- **Break-even** : en combien d'années le canal est rentable ?

## Pourquoi ça matters

L'argument "baisser la mer" seul est marginal (quelques mm). L'argument "créer des civilisations dans le désert + baisser la mer + être rentable" est transformateur. C'est la différence entre un projet de niche scientifique et un projet politique/économique viable.

## Complexité technique

- Modèles économiques + timeline : moyen (paramètres connus dans la littérature)
- Spiruline/aquaculture : requires domain research (rendements, prix marché)
- Réseau ramifié multi-canaux : complexe (nouvelle architecture UI)
- Valeur foncière zones habitables : complexe (dépend géographie, pays)
